# Tarea 1 - Fundamentos de la Web

Este proyecto contiene la estructura básica de un sitio web utilizando HTML5, CSS y JavaScript, cumpliendo con los requisitos de la práctica de la universidad.

## Estructura del Proyecto
- `index.html`: Estructura principal, etiquetas semánticas y contenido.
- `css/styles.css`: Estilos visuales, colores y formato.
- `js/script.js`: Interactividad básica y alertas.

**¿Qué fue lo más difícil?**
Definitivamente lo más difícil fue la última parte de subir los archivos a GitHub. Me salieron mil errores en la pantalla negra (la terminal): primero que no reconocía Git, luego problemas con mi identidad de usuario, después que no encontraba el repositorio y hasta me rechazaba la subida final. Fue una pelea constante con la consola para lograr que suba.

**¿Qué no entendieron completamente?**
La verdad, no entendí mucho de esa última parte de los comandos de Git. Logré ejecutar los pasos para hacer los *commits* y hasta tuve que forzar el *push* para arreglar los errores y que suba de una vez, pero la lógica exacta de cómo funciona esa conexión por debajo entre mi computadora y la nube todavía me resulta bastante confusa.

**¿Para qué creen que sirve Git en proyectos reales?**
Es como tener un sistema de "guardar partida" para nuestro código. En proyectos reales, sirve para tener un historial seguro de todo lo que hacemos. Si nos equivocamos o rompemos algo en el proyecto, podemos volver a una versión anterior sin perder semanas de trabajo, además de que permite a los equipos trabajar juntos sin borrar los archivos del otro.

**¿Por qué es importante estructurar bien un proyecto?**
Es vital para mantener el orden. Si dejamos todos los archivos sueltos (el HTML, el CSS y el JS mezclados en el mismo lugar), cuando el proyecto crezca va a ser imposible encontrar un error. Una buena estructura con carpetas separadas hace que el código sea más ordenado, fácil de leer y sencillo de revisar.
